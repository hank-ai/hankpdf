# v0.3.0 Mega-Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship hankpdf 0.3.0 with production-safety hardening (cross-platform per-page worker memory caps, decompression-bomb regression corpus, signed-PDF preservation flag), removal of the `pdf_smasher/` deprecation shim, lazy native-dep boot check, and small API debt cleanup.

**Architecture:** Layer the new safety code on top of the existing pipeline. New module `hankpdf/_environment.py` owns native-dep probing and floor enforcement; existing `hankpdf.exceptions.EnvironmentError` is reused. New module `hankpdf/sandbox/platform_caps.py` (replacing today's 8-line stub) implements `apply_self_memory_cap` via `resource.setrlimit(RLIMIT_AS)` on Unix and ctypes-against-`kernel32.dll` Job Object self-assign on Windows ≥ 8 (no pywin32). The existing `_init_worker` ProcessPoolExecutor initializer in `hankpdf/__init__.py` gains a positional `mem_cap: int` argument and applies the cap inside each worker. A psutil-backed RSS watchdog runs in a parent-side thread as redundant backstop. Signed-PDF default-refusal is already present; this release adds an explicit `--preserve-signatures` opt-in for verbatim passthrough plus two new CompressReport fields (schema v4 → v5, additive).

**Tech Stack:** Python 3.14, hatch build backend, pikepdf, pypdfium2, opencv, Pillow, pytest. New runtime dep `psutil>=5.9`. New test-only dep `pyhanko>=0.27`. Windows uses ctypes against `kernel32.dll` directly. Linux/macOS uses stdlib `resource`.

**Spec:** `docs/superpowers/specs/2026-05-02-v0.3.0-mega-release-design.md` (commit `8ef5d69`).

**Branch:** `feat/v0.3.0-mega-release` (already created).

## Plan Revisions (post-/dc review, 2026-05-02)

A double-check planning review surfaced 4 CRITICAL and ~15 MEDIUM findings. Fixes applied inline below. Summary:

**CRITICAL fixes applied to the plan:**

1. **Task 12 extended to `docs/THREAT_MODEL.md`.** §1 currently states RLIMIT_AS / Job Object are "planned, not implemented in this build" — this release closes that gap. Footer Last-Reviewed bumped to 2026-05-02.
2. **Memory cap heuristic hard-clamped.** `_compute_worker_mem_cap` is now `min(max(8 GB, 16 × input_size), 16 GB)`. The CLI side already enforces `--max-input-mb=250` via `_enforce_input_policy`, but the library API surface had no clamp; a hostile 4 GB caller could lift the cap to 64 GB. Hard ceiling + spec §4.D documentation.
3. **Watchdog cooperative shutdown.** Plan no longer SIGTERMs workers mid-write — that path corrupts partial pikepdf output streams. New design: per-worker `multiprocessing.Event` (`abort_event`) the worker checks at safe-write boundaries inside `_process_single_page`; worker drains current chunk and exits cleanly. SIGKILL is the kernel's job (RLIMIT_AS / Job Object); the watchdog only requests cooperative shutdown.
4. **Aggregate cap envelope.** Before constructing the executor, `compress()` computes `effective_cap = min(per_worker_cap, (psutil.virtual_memory().available × 0.7) / n_workers)`. If the chosen caps would exceed 70% of host RAM in aggregate, `n_workers` is reduced (or the run aborts with a structured warning). Default `n_workers` already auto-clamps to `cpu_count - 2`; this adds RAM-pressure clamping.

**MEDIUM fixes applied:**

- Tightened `psutil>=5.9,<8` upper bound (Task 0). Renovate/Dependabot upgrades won't sweep customers onto an untested major.
- Robust `_probe()` (Task 1): broader `Exception` catch, byte-bounded `Popen` reads (16 KB cap on stdout/stderr), graceful UnicodeDecodeError handling. Pillow's chatty OpenJPEG fork can't deadlock the probe.
- `correlation_id` validation moved BEFORE `assert_environment_ready()` (Task 2). Argument-validation failures are more proximate than env-precondition failures; user gets the right error.
- `[W-CAPS-FAILED]` and `[W-CAPS-UNAVAILABLE]` now flow into `CompressReport.warnings` (not just stderr) — library callers can detect that the cap is not active. Added to schema-v5 contract (Task 9).
- Watchdog `except` list extended to include `psutil.ZombieProcess`. Watchdog liveness check in parent's `finally`; new `[W-WATCHDOG-DIED]` warning code. Detect kernel-cap kill via worker `exitcode` (SIGKILL=-9 on Unix; `STATUS_QUOTA_EXCEEDED=0xC0000044` on Windows) — not via watchdog flag — so kernel-kill paths surface as `MemoryCapExceededError` (exit 18), not as `EXIT_ENGINE_ERROR=30`.
- `_PreserveSignaturesSignal` refactored to a `PolicyDecision` enum return (Task 9). The exception sentinel was leaking through `_run_image_export` which also calls `_enforce_input_policy`. Returning a value is mypy-friendly and survives wrapper refactors.

- Bomb test `expected_codes` tuples are pinned to ONE code per fixture (Task 10). If today's engine produces `EXIT_ENGINE_ERROR=30` for a fixture, that's the bug the corpus exists to catch — file follow-up to wire the right structured refusal in 0.3.x.
- Task 0 adds license verification step (`uv pip licenses` for `psutil` and `pyhanko`); both are MIT/BSD-3 (verified).
- Task 11 (ROADMAP rename) extended: `mode=medical` → `mode=strict` AND grep the rest of the docs for any stray "medical" references.
- `[W-WATCHDOG-DIED]` added to `CliWarningCode` Literal in Task 13.

**FALSE POSITIVE — DO NOT FIX:**

- The main reviewer flagged `except AssertionError, CompressError:` at `hankpdf/__init__.py:1221, :1280` and `cli/main.py:604` as a syntax bug ("parses as `except A as B:`"). **PEP 758 (Python 3.14) explicitly permits unparenthesized `except A, B:` with multiple alternatives.** Verified by Apr 25 checkpoint + multiple prior DCR rounds: this is a recurring DCR false positive. Probed once per process (cached via ``functools.cache``). Library users
```python
pay the subprocess cost only when they call :func:`hankpdf.compress`;
the CLI invokes the same check eagerly at argparse-completion.

intentionally run with stub Tesseract / qpdf binaries.

Floors enforced
---------------
- Python ≥ 3.14 (already enforced by ``pyproject.toml``)
- Tesseract ≥ 5.0.0 (tessdata format)
- qpdf ≥ 11.6.3 (qpdf #1050: silent character-drop in 11.0–11.6.2)
- OpenJPEG ≥ 2.5.4 (CVE-2025-54874 OOB heap write); probed via Pillow

_VERSION_RE = re.compile(r"\b(\d+\.\d+(?:\.\d+)?)\b")


@dataclass(frozen=True, slots=True)
class EnvFailure:
    """One reason hankpdf cannot start."""

    component: str  # "tesseract" | "qpdf" | "jbig2enc" | "openjpeg"
    reason: str  # "missing" | "below-floor" | "unparseable-version"

def parse_tesseract_version(raw: str) -> str | None:
    """Tesseract prints ``tesseract 5.3.4`` on the first line."""
    first = (raw or "").splitlines()[:1]
    if not first:
        return None
    m = _VERSION_RE.search(first[0])
    return m.group(1) if m else None


def parse_qpdf_version(raw: str) -> str | None:
    """qpdf prints ``qpdf version 11.6.4`` on the first line."""
    first = (raw or "").splitlines()[:1]
    if not first:
        return None
    m = _VERSION_RE.search(first[0])
    return m.group(1) if m else None


def parse_jbig2_version(raw: str) -> str | None:
    """jbig2enc prints ``jbig2enc 0.29`` on the first line; sometimes via stderr."""
    first = (raw or "").splitlines()[:1]
    if not first:
        return None
    m = _VERSION_RE.search(first[0])
    return m.group(1) if m else None


def _version_tuple(v: str) -> tuple[int, ...]:
    return tuple(int(p) for p in v.split(".") if p.isdigit())


def _meets_floor(found: str, floor: str) -> bool:
    return _version_tuple(found) >= _version_tuple(floor)


# ----------------------------------------------------------------------
# Per-tool probes
# ----------------------------------------------------------------------


_PROBE_BUFFER_CAP = 16 * 1024  # 16 KB is more than enough for any --version


def _probe(tool: str, parser) -> tuple[str, str | None] | None:  # noqa: ANN001
    """Probe a native tool's version with bounded output and broad failure tolerance.

    Bounded reads matter: Pillow's bundled OpenJPEG fork (and a few other
    packages) emit large feature tables on stderr when invoked with
    --version. Default capture_output=True buffers the entire output in
    memory until process exit; a verbose tool can balloon the parent's
    RSS or block on a filled pipe buffer.

    Broad except: text=True can raise UnicodeDecodeError when a tool
    writes non-UTF-8 output (real failure mode on Windows builds with
    non-ASCII install paths). subprocess.SubprocessError covers the
    timeout / process-error family. We treat any failure as
    (path, None) — "tool present but version unparseable" — so the
    boot check doesn't crash on an unexpected failure mode.
    """
    path = shutil.which(tool)
    if path is None:
        return None
    try:
        proc = subprocess.Popen(
            [tool, "--version"],
            stdout=subprocess.PIPE,
            stderr=subprocess.PIPE,
        )
        try:
            stdout, stderr = proc.communicate(timeout=5)
        except subprocess.TimeoutExpired:
            proc.kill()
            proc.communicate()
            return (path, None)
        # Truncate to bounded buffer; a >16KB --version means something
        # weird is going on, and the version string itself is in the
        # first ~100 bytes.
        stdout = stdout[:_PROBE_BUFFER_CAP] if stdout else b""
        stderr = stderr[:_PROBE_BUFFER_CAP] if stderr else b""
        # decode with errors="replace" — non-UTF-8 paths are real on Windows
        raw = (stdout or stderr).decode("utf-8", errors="replace")
    except Exception:
        return (path, None)
    return (path, parser(raw))

    and parses the version strings. Subsequent calls return the cached
    :class:`EnvironmentReport` instance.

    Set ``HANKPDF_SKIP_ENV_CHECK=1`` to short-circuit and return an empty
    report (no failures). Used by tests that intentionally run without
    a real Tesseract/qpdf binary.
    """
    if os.environ.get("HANKPDF_SKIP_ENV_CHECK") == "1":
        return EnvironmentReport(
            python_version=sys.version.split()[0],
            platform=_platform.platform(),
            tesseract=None,
            qpdf=None,
            jbig2enc=None,
            openjpeg_via_pillow=None,
            pdfium_revision=None,
            pillow_max_image_pixels=_pillow_max(),
            failures=(),
        )

    failures: list[EnvFailure] = []

    # Tesseract
    t = _probe("tesseract", parse_tesseract_version)
    if t is None:
        failures.append(
            EnvFailure(
                component="tesseract",
                reason="missing",
                found=None,
                required=TESSERACT_FLOOR,
                install_hint=_install_hint_tesseract(),
            )

    else:

        if version is None:
            failures.append(
                EnvFailure(
                    component="tesseract",
                    reason="unparseable-version",
                    found=None,
                    required=TESSERACT_FLOOR,
                    install_hint=_install_hint_tesseract(),
                )
            )
        elif not _meets_floor(version, TESSERACT_FLOOR):
            failures.append(
                EnvFailure(
                    component="tesseract",
                    reason="below-floor",
                    found=version,
                    required=TESSERACT_FLOOR,
                    install_hint=_install_hint_tesseract(),
                )
            )

    q = _probe("qpdf", parse_qpdf_version)
    if q is None:
        failures.append(
            EnvFailure(
                component="qpdf",
                reason="missing",
                found=None,
                required=QPDF_FLOOR,
                install_hint=_install_hint_qpdf(),
            )
        )
    else:
        _, version = q
        if version is None:
            failures.append(
                EnvFailure(
                    component="qpdf",
                    reason="unparseable-version",
                    found=None,
                    required=QPDF_FLOOR,
                    install_hint=_install_hint_qpdf(),
                )
            )
        elif not _meets_floor(version, QPDF_FLOOR):
            failures.append(
                EnvFailure(
                    component="qpdf",
                    reason="below-floor",
                    found=version,
                    required=QPDF_FLOOR,
                    install_hint=_install_hint_qpdf(),
                )
            )

    j = _probe("jbig2", parse_jbig2_version)

    # OpenJPEG via Pillow — encode probe.
    openjpeg_status = _probe_openjpeg()

        python_version=sys.version.split()[0],
        platform=_platform.platform(),
        tesseract=t,
        qpdf=q,
        jbig2enc=j,
        openjpeg_via_pillow=openjpeg_status,
        pdfium_revision=_pdfium_rev(),
        pillow_max_image_pixels=_pillow_max(),
        failures=tuple(failures),
    )


def _probe_openjpeg() -> str | None:
    """Encode an 8x8 RGB image to JPEG2000 to verify Pillow's bundled
    OpenJPEG. Returns the version string Pillow reports, or ``"available"``
    when the version isn't surfaced.
    """
    try:
        import io

        from PIL import Image, features
    except ImportError:
        return None
    try:
        Image.new("RGB", (8, 8)).save(io.BytesIO(), format="JPEG2000")
    except (OSError, RuntimeError):
        return None
    try:
        return features.version("jpg_2000") or "available"
    except (KeyError, ValueError):
        return "available"


def _pillow_max() -> int:
    try:
        from PIL import Image

        return int(Image.MAX_IMAGE_PIXELS or 0)
    except ImportError:
        return 0


def _pdfium_rev() -> str | None:
    try:
        import pypdfium2 as _p

        return str(getattr(_p, "PDFIUM_INFO", "") or getattr(_p, "V_LIBPDFIUM", "") or "")
    except (ImportError, AttributeError):
        return None


def assert_environment_ready() -> EnvironmentReport:
    """Probe the environment and raise if anything is below the floor.

    Returns the :class:`EnvironmentReport` so callers can inspect it
    after a successful check (e.g., to log the resolved versions). The
    raised :class:`EnvironmentError` carries the full ``failures`` tuple
    for programmatic handling.
    """
    report = get_environment_report()
    if report.failures:
        message = format_failure_message(report.failures)
        exc = EnvironmentError(message)
        exc.failures = report.failures  # type: ignore[attr-defined]
        raise exc
    return report


def format_failure_message(failures: tuple[EnvFailure, ...]) -> str:
    """Multi-line human-readable failure message printed by the CLI."""
    if not failures:
        return ""
    lines = [
        "hankpdf cannot start. Required native dependencies are missing or below the supported floor:",
        "",
    ]
    for f in failures:
        if f.reason == "missing":
            lines.append(f"  {f.component:10s} : not found in PATH")
        elif f.reason == "below-floor":
            lines.append(
                f"  {f.component:10s} : {f.found} found, {f.required} required"
            )
        else:
            lines.append(f"  {f.component:10s} : {f.reason}")
        for hint in f.install_hint.splitlines():
            lines.append(f"               {hint}")
        lines.append("")
    lines.append("Run `hankpdf --doctor` for the full report.")
    return "\n".join(lines)
```

```python
def compress_stream(
    input_stream: IO[bytes],
    output_stream: IO[bytes],
    options: CompressOptions | None = None,
    *,
    correlation_id: str | None = None,
) -> CompressReport:
    """Streaming compression variant. See ``docs/SPEC.md`` §1.2.

    ``correlation_id`` (optional, ``[A-Za-z0-9._:-]{1,64}``) is stamped
    onto the returned :class:`CompressReport`. Pass the same id your
    upstream logger prefixes its lines with so the report can be
    joined back to the calling system's log slice. If omitted, a fresh
    UUID4 hex is generated.
    """
    data = input_stream.read()
    out, report = compress(data, options=options, correlation_id=correlation_id)
    output_stream.write(out)
    return report
```

```python
"""Per-page MRC gate.

Decides which pages go through the MRC pipeline vs verbatim passthrough,
based on the per-page image-byte fraction. Lifted out of
``hankpdf/__init__.py`` in v0.3.0 so the 1455-line module stops growing.

The gate is a pre-filter, not a verifier. Conservative biases (no nested
Form XObject walk, no parent-inherited /Resources) push toward MRC, the
safe direction. See SPEC.md §4.1b.
"""

from __future__ import annotations

from dataclasses import dataclass

from hankpdf.engine.page_classifier import score_pages_for_mrc
from hankpdf.types import CompressOptions, TriageReport


@dataclass(frozen=True, slots=True)
class PerPageGateResult:
    """Outcome of the per-page MRC gate.

    Fields:
        mrc_worthy: per-page bool. True → run MRC; False → verbatim copy.
        whole_doc_passthrough: True iff no page met the threshold AND
            no override (re_ocr / strip_text_layer / legal_codec_profile /
            verify) forces the full pipeline. Caller returns input bytes
            unchanged when this is set.
        warnings: structured warning codes recorded into CompressReport.
    """

    mrc_worthy: tuple[bool, ...]
    whole_doc_passthrough: bool
    warnings: tuple[str, ...]


def _force_full_pipeline(options: CompressOptions) -> bool:
    """Mirror the existing interlock from compress(): any of these flags
    forces every page through the MRC pipeline.
    """
    return bool(
        options.re_ocr
        or options.strip_text_layer
        or options.legal_codec_profile
        or not options.skip_verify
    )


def run_per_page_gate(
    input_data: bytes,
    triage: TriageReport,
    options: CompressOptions,
) -> PerPageGateResult:
    """Score pages, decide passthrough vs MRC, return the gate result.

    The caller threads ``mrc_worthy`` into ``_WorkerInput.mrc_worthy``
    per page. If ``whole_doc_passthrough`` is True, the caller emits the
    ``passthrough-no-image-content`` warning and returns input bytes
    unchanged.

    Note: ``score_pages_for_mrc`` takes raw bytes and re-opens via
    pikepdf internally. Reusing the caller's already-open Pdf would be
    a different refactor; staying byte-based preserves call semantics.
    """
    if _force_full_pipeline(options):
        return PerPageGateResult(
            mrc_worthy=tuple([True] * triage.pages),
            whole_doc_passthrough=False,
            warnings=(),
        )

    if options.min_image_byte_fraction <= 0.0:
        return PerPageGateResult(
            mrc_worthy=tuple([True] * triage.pages),
            whole_doc_passthrough=False,
            warnings=(),
        )

    flags = score_pages_for_mrc(
        input_data,
        password=options.password,
        min_image_byte_fraction=options.min_image_byte_fraction,
    )

    # Defensive: triage is the source of truth for page count downstream.
    if len(flags) != triage.pages:
        flags = list(flags) + [True] * max(0, triage.pages - len(flags))
        flags = flags[: triage.pages]

    if not any(flags):
        return PerPageGateResult(
            mrc_worthy=tuple(flags),
            whole_doc_passthrough=True,
            warnings=("passthrough-no-image-content",),
        )

    skipped = [i for i, w in enumerate(flags) if not w]
    warnings: list[str] = []
    if skipped:
        warnings.append(f"pages-skipped-verbatim-{len(skipped)}")

    return PerPageGateResult(
        mrc_worthy=tuple(flags),
        whole_doc_passthrough=False,
        warnings=tuple(warnings),
    )
```

```python
"""Cross-platform per-process memory cap.

``apply_self_memory_cap(bytes)`` applies a hard memory cap to the
*current* process. Designed to be called from inside a
ProcessPoolExecutor worker via the ``initializer`` hook. When the worker
exceeds the cap, the OS terminates the process; the parent's
``concurrent.futures`` driver surfaces the broken pipe as a normal task
failure that callers can translate into MemoryCapExceededError.

Linux/macOS uses RLIMIT_AS. Windows uses ctypes against kernel32.dll to
self-assign to a Job Object with JOB_OBJECT_LIMIT_PROCESS_MEMORY (Win 8+).
No pywin32 dep.
"""

from __future__ import annotations

import sys


class CapsUnavailableError(RuntimeError):
    """Raised when no memory-cap primitive is available on this OS."""


def supported() -> bool:
    """Return True if apply_self_memory_cap will succeed on this platform."""
    return sys.platform in ("linux", "darwin", "win32")


def apply_self_memory_cap(byte_limit: int) -> None:
    """Apply a per-process memory cap. ``byte_limit=0`` disables (no-op).

    Raises:
        ValueError: byte_limit < 0.
        CapsUnavailableError: this OS doesn't support self-capping.
        OSError: kernel rejected the call.
    """
    if byte_limit < 0:
        msg = f"byte_limit must be >= 0 (got {byte_limit})"
        raise ValueError(msg)
    if byte_limit == 0:
        return  # disabled
    if sys.platform in ("linux", "darwin"):
        from hankpdf.sandbox._apply_unix import apply

        apply(byte_limit)
        return
    if sys.platform == "win32":
        from hankpdf.sandbox._apply_win32 import apply  # provided in Task 7

        apply(byte_limit)
        return
    msg = f"no memory-cap primitive available on platform {sys.platform!r}"
    raise CapsUnavailableError(msg)
```

```python
"""Windows Job Object self-assign for per-process memory caps.

Uses ctypes against kernel32.dll directly — no pywin32 dependency.
Supported on Windows 8+ (nested Job Objects). hankpdf already requires
Python 3.14, which dropped Windows 7/8 upstream, so this matches the
supported platform floor.

Mechanism:

1. CreateJobObjectW(NULL, NULL) -> hJob.
2. SetInformationJobObject with JobObjectExtendedLimitInformation
   carrying ProcessMemoryLimit and the JOB_OBJECT_LIMIT_PROCESS_MEMORY
   + JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE flags.
3. AssignProcessToJobObject(hJob, GetCurrentProcess()).
4. Intentionally leak hJob: closing it would kill us due to
   KILL_ON_JOB_CLOSE. The kernel releases the handle when the
   process exits.
"""

from __future__ import annotations

import ctypes
import ctypes.wintypes as wt


class _JOBOBJECT_BASIC_LIMIT_INFORMATION(ctypes.Structure):
    _fields_ = [
        ("PerProcessUserTimeLimit", ctypes.c_int64),
        ("PerJobUserTimeLimit", ctypes.c_int64),
        ("LimitFlags", wt.DWORD),
        ("MinimumWorkingSetSize", ctypes.c_size_t),
        ("MaximumWorkingSetSize", ctypes.c_size_t),
        ("ActiveProcessLimit", wt.DWORD),
        ("Affinity", ctypes.c_void_p),
        ("PriorityClass", wt.DWORD),
        ("SchedulingClass", wt.DWORD),
    ]


class _IO_COUNTERS(ctypes.Structure):
    _fields_ = [
        ("ReadOperationCount", ctypes.c_uint64),
        ("WriteOperationCount", ctypes.c_uint64),
        ("OtherOperationCount", ctypes.c_uint64),
        ("ReadTransferCount", ctypes.c_uint64),
        ("WriteTransferCount", ctypes.c_uint64),
        ("OtherTransferCount", ctypes.c_uint64),
    ]


class _JOBOBJECT_EXTENDED_LIMIT_INFORMATION(ctypes.Structure):
    _fields_ = [
        ("BasicLimitInformation", _JOBOBJECT_BASIC_LIMIT_INFORMATION),
        ("IoInfo", _IO_COUNTERS),
        ("ProcessMemoryLimit", ctypes.c_size_t),
        ("JobMemoryLimit", ctypes.c_size_t),
        ("PeakProcessMemoryUsed", ctypes.c_size_t),
        ("PeakJobMemoryUsed", ctypes.c_size_t),
    ]


_JOB_OBJECT_LIMIT_PROCESS_MEMORY = 0x100
_JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE = 0x2000
_JobObjectExtendedLimitInformation = 9


def apply(byte_limit: int) -> None:
    """Self-assign the current process to a Job Object with a memory cap."""
    kernel32 = ctypes.WinDLL("kernel32", use_last_error=True)

    kernel32.CreateJobObjectW.restype = wt.HANDLE
    kernel32.CreateJobObjectW.argtypes = [ctypes.c_void_p, wt.LPCWSTR]
    h_job = kernel32.CreateJobObjectW(None, None)
    if not h_job:
        raise OSError(ctypes.get_last_error(), "CreateJobObjectW")

    info = _JOBOBJECT_EXTENDED_LIMIT_INFORMATION()
    info.BasicLimitInformation.LimitFlags = (
        _JOB_OBJECT_LIMIT_PROCESS_MEMORY | _JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE
    )
    info.ProcessMemoryLimit = ctypes.c_size_t(byte_limit)

    kernel32.SetInformationJobObject.restype = wt.BOOL
    kernel32.SetInformationJobObject.argtypes = [
        wt.HANDLE,
        ctypes.c_int,
        ctypes.c_void_p,
        wt.DWORD,
    ]
    if not kernel32.SetInformationJobObject(
        h_job,
        _JobObjectExtendedLimitInformation,
        ctypes.byref(info),
        ctypes.sizeof(info),
    ):
        raise OSError(ctypes.get_last_error(), "SetInformationJobObject")

    kernel32.GetCurrentProcess.restype = wt.HANDLE
    kernel32.GetCurrentProcess.argtypes = []
    h_proc = kernel32.GetCurrentProcess()

    kernel32.AssignProcessToJobObject.restype = wt.BOOL
    kernel32.AssignProcessToJobObject.argtypes = [wt.HANDLE, wt.HANDLE]
    if not kernel32.AssignProcessToJobObject(h_job, h_proc):
        raise OSError(ctypes.get_last_error(), "AssignProcessToJobObject")
    # Intentionally leak h_job — closing kills us via KILL_ON_JOB_CLOSE.
```

```python
import psutil

# Hard ceiling. Even for legitimate huge scans (rasterized bombs are bounded
# by Pillow MAX_IMAGE_PIXELS upstream), 16 GB per worker is more than enough.
_HARD_CEILING_BYTES = 16 * 1024**3
# Floor — ensures small inputs still get a generous cap.
_FLOOR_BYTES = 8 * 1024**3
# Per-page raster inflation factor over input bytes (300 DPI rasterization).
_INFLATION_FACTOR = 16


def _compute_worker_mem_cap(
    input_size_bytes: int,
    n_workers: int,
    options: CompressOptions,
) -> int:
    """Compute the per-worker memory cap.

    Three constraints, in order of precedence:

    1. Aggregate envelope: cap × n_workers must not exceed 70% of host
       available RAM. We pick the smaller of (per-worker computed cap)
       and (host-available × 0.7 / n_workers). Avoids the 7 × 8 GB =
       56 GB scenario on a 32 GB host.
    2. Hard ceiling: never more than 16 GB per worker, regardless of
       input size. A library caller passing 4 GB of bytes would
       otherwise lift the cap to 64 GB (16 × 4 GB) — defeats the bomb
       defense for that input. The CLI clamps via --max-input-mb=250
       but the library API has no such clamp, hence this absolute cap.
    3. Per-input scaling: max(floor, inflation × input_size). Floor is
       8 GB so trivial inputs still get headroom for legitimate
       rasterization.

    options.max_worker_memory_mb overrides constraints 2 and 3 (caller
    knows their workload); constraint 1 (aggregate envelope) still
    applies as a host-protection floor.
    """
    if options.max_worker_memory_mb is not None:
        # 0 is the documented test escape hatch; bypass all clamps.
        if options.max_worker_memory_mb == 0:
            return 0
        per_worker = int(options.max_worker_memory_mb) * 1024 * 1024
    else:
        per_worker = max(_FLOOR_BYTES, _INFLATION_FACTOR * input_size_bytes)
        per_worker = min(per_worker, _HARD_CEILING_BYTES)

    # Aggregate envelope — protect the host.
    try:
        available = psutil.virtual_memory().available
    except Exception:
        # If psutil can't read /proc on a sealed container, skip the
        # envelope check. The hard ceiling above still applies.
        return per_worker
    aggregate_budget = int(available * 0.7)
    envelope_per_worker = aggregate_budget // max(1, n_workers)
    return min(per_worker, envelope_per_worker)
```

- [ ] **Step 7: Add the parent-side RSS watchdog (cooperative, liveness-checked)**

Inside `compress()`, near where the `ProcessPoolExecutor` is created and futures are submitted, spawn a watchdog thread. Two key revisions from the original plan:

- **Cooperative shutdown.** The original plan called `psutil.Process(pid).terminate()` (SIGTERM on Unix; TerminateProcess on Windows) which can interrupt a worker mid-`pikepdf.save()` and produce partial bytes that get merged into the final output. The kernel cap (RLIMIT_AS / Job Object) is the authoritative kill — the watchdog only requests cooperative shutdown via a per-worker `multiprocessing.Event`. Workers check the event at safe-write boundaries (before each `pikepdf.save()` chunk) and exit cleanly.
- **Liveness check.** A daemon thread that dies from an unhandled exception (e.g., `psutil.ZombieProcess` on a heavily-loaded host) goes silently deaf. The parent's `finally` block now asserts `watchdog.is_alive()` and emits `[W-WATCHDOG-DIED]` if not.

```python
import threading
import psutil

class _WatchdogState:
    """Watchdog state object — readable by the result-collection loop
    after the watchdog has exited. ``exitcodes`` is populated as workers
    disappear from the executor's process table; this avoids the race
    where ``BrokenProcessPool`` clears ``_processes`` before
    ``_classify_worker_death`` reads it.
    """
    __slots__ = (
        "any_cap_exceeded",
        "thread_died",
        "stop_event",
        "exitcodes",
    )
    def __init__(self) -> None:
        self.any_cap_exceeded: bool = False
        self.thread_died: bool = False
        self.stop_event: threading.Event = threading.Event()
        self.exitcodes: dict[int, int] = {}


def _start_rss_watchdog(
    executor: ProcessPoolExecutor,
    mem_cap: int,
    abort_event,  # mp.Event, shared with all workers
    state: _WatchdogState,
) -> threading.Thread:
    """Poll each worker's RSS; if any exceeds the cap, set the shared
    abort_event so ALL workers cooperatively exit at the next
    safe-write boundary.

    The kernel-level cap (RLIMIT_AS on Unix, Job Object on Windows) is
    the authoritative SIGKILL. This watchdog catches malloc-by-mmap
    escapes from RLIMIT_AS on Linux and provides observability for all
    platforms. It does NOT call psutil.Process.terminate() — that path
    corrupts partial output streams.

    Cost: one psutil call per worker per 500ms. Negligible on idle
    hosts; under heavy load /proc reads can take >10ms each. The
    watchdog reads under the GIL (so does the main thread) but the
    polling is bounded by the loop sleep.

    Also proactively snapshots worker exitcodes as workers exit, so the
    result-collection loop can classify deaths even after
    BrokenProcessPool clears the executor's _processes table.
    """
    if mem_cap <= 0:
        # No cap: return a stub thread so the finally-block join is safe.
        t = threading.Thread(target=lambda: None, daemon=True)
        t.start()
        return t

    def _loop() -> None:
        try:
            seen_pids: set[int] = set()
            while not state.stop_event.wait(0.5):
                processes = dict(getattr(executor, "_processes", {}))
                for pid, proc in processes.items():
                    seen_pids.add(pid)
                    # Snapshot exitcode if the worker has died; do this
                    # before the RSS poll so we capture deaths the poll
                    # missed.
                    ec = getattr(proc, "exitcode", None)
                    if ec is not None and pid not in state.exitcodes:
                        state.exitcodes[pid] = ec
                    try:
                        rss = psutil.Process(pid).memory_info().rss
                    except (psutil.NoSuchProcess, psutil.AccessDenied,
                            psutil.ZombieProcess):
                        continue
                    if rss > mem_cap:
                        sys.stderr.write(
                            f"[W-MEM-RSS-WATCHDOG] worker pid={pid} "
                            f"RSS={rss} exceeded cap={mem_cap}; "
                            f"requesting cooperative shutdown\n"
                        )
                        # Set the shared event — ALL workers see it and
                        # cooperatively drain at next safe-write boundary.
                        abort_event.set()
                        state.any_cap_exceeded = True
                # Capture exitcodes for workers that left the table.
                for pid in list(seen_pids - set(processes.keys())):
                    if pid not in state.exitcodes:
                        # Best-effort — table may not have entries for
                        # workers that already cleaned up.
                        state.exitcodes.setdefault(pid, -1)
        except Exception as e:  # never let the watchdog die silently
            state.thread_died = True
            sys.stderr.write(f"[W-WATCHDOG-DIED] {type(e).__name__}: {e}\n")
            raise

    t = threading.Thread(target=_loop, daemon=True, name="hankpdf-rss-watchdog")
    t.start()
    return t


# Where the executor is created (Step 5 above):
state = _WatchdogState()

try:
```

```python
"""Generate signed-PDF fixtures for tests/corpus/signed/.

Test-only — uses pyhanko (test extra dep).

Outputs:
  simple_signed.pdf       — single approval signature
  certifying_signed.pdf   — DocMDP certifying signature (no changes allowed)
  multi_signed.pdf        — two approval signatures

Run: python scripts/generate_signed_corpus.py tests/corpus/signed/
"""

from __future__ import annotations

import argparse
import io
from pathlib import Path

import pikepdf
from pyhanko.pdf_utils.incremental_writer import IncrementalPdfFileWriter
from pyhanko.sign import PdfSignatureMetadata, SimpleSigner, signers
from pyhanko_certvalidator.registry import SimpleCertificateStore


def _empty_pdf() -> bytes:
    pdf = pikepdf.new()
    pdf.add_blank_page(page_size=(612, 792))
    buf = io.BytesIO()
    pdf.save(buf)
    return buf.getvalue()


def _self_signed_signer(name: str) -> SimpleSigner:
    """Generate a self-signed cert in-memory for fixture purposes."""
    from cryptography import x509
    from cryptography.hazmat.primitives import hashes, serialization
    from cryptography.hazmat.primitives.asymmetric import rsa
    from datetime import datetime, timedelta, timezone

    key = rsa.generate_private_key(public_exponent=65537, key_size=2048)
    subject = issuer = x509.Name([x509.NameAttribute(x509.NameOID.COMMON_NAME, name)])
    now = datetime.now(timezone.utc)
    cert = (
        x509.CertificateBuilder()
        .subject_name(subject)
        .issuer_name(issuer)
        .public_key(key.public_key())
        .serial_number(x509.random_serial_number())
        .not_valid_before(now)
        .not_valid_after(now + timedelta(days=365))
        .sign(key, hashes.SHA256())
    )
    return SimpleSigner(
        signing_cert=cert,
        signing_key=key,
        cert_registry=SimpleCertificateStore(),
    )


def make_simple_signed(out: Path) -> None:
    src = io.BytesIO(_empty_pdf())
    writer = IncrementalPdfFileWriter(src)
    signer = _self_signed_signer("hankpdf test signer")
    signers.PdfSigner(
        PdfSignatureMetadata(field_name="Sig1"), signer=signer
    ).sign_pdf(writer, output=out)


def make_certifying_signed(out: Path) -> None:
    src = io.BytesIO(_empty_pdf())
    writer = IncrementalPdfFileWriter(src)
    signer = _self_signed_signer("hankpdf test certifier")
    signers.PdfSigner(
        PdfSignatureMetadata(field_name="Sig1", certify=True),
        signer=signer,
    ).sign_pdf(writer, output=out)


def make_multi_signed(out: Path) -> None:
    src = io.BytesIO(_empty_pdf())
    writer = IncrementalPdfFileWriter(src)
    signer = _self_signed_signer("hankpdf test signer A")
    intermediate = io.BytesIO()
    signers.PdfSigner(
        PdfSignatureMetadata(field_name="Sig1"), signer=signer
    ).sign_pdf(writer, output=intermediate)
    intermediate.seek(0)
    writer = IncrementalPdfFileWriter(intermediate)
    signer = _self_signed_signer("hankpdf test signer B")
    signers.PdfSigner(
        PdfSignatureMetadata(field_name="Sig2"), signer=signer
    ).sign_pdf(writer, output=out)


def main() -> None:
    p = argparse.ArgumentParser()
    p.add_argument("out_dir", type=Path)
    args = p.parse_args()
    args.out_dir.mkdir(parents=True, exist_ok=True)
    make_simple_signed(args.out_dir / "simple_signed.pdf")
    make_certifying_signed(args.out_dir / "certifying_signed.pdf")
    make_multi_signed(args.out_dir / "multi_signed.pdf")
    print(f"wrote 3 signed-PDF fixtures to {args.out_dir}")


if __name__ == "__main__":
    main()
```

```python
"""Integration: signed-PDF default-refuse + --preserve-signatures + --allow-signed-invalidation."""

from __future__ import annotations

import subprocess
import sys
from pathlib import Path

import pytest

CORPUS = Path("tests/corpus/signed")


@pytest.fixture(scope="session", autouse=True)
def _ensure_corpus() -> None:
    expected = ["simple_signed.pdf", "certifying_signed.pdf", "multi_signed.pdf"]
    missing = [n for n in expected if not (CORPUS / n).exists()]
    if missing:
        pytest.skip(f"signed-PDF corpus missing: {missing} — run scripts/generate_signed_corpus.py")


def _run_cli(args: list[str], env_extra: dict | None = None) -> subprocess.CompletedProcess:
    import os

    env = os.environ.copy()
    env["HANKPDF_SKIP_ENV_CHECK"] = "1"
    if env_extra:
        env.update(env_extra)
    return subprocess.run(
        [sys.executable, "-m", "hankpdf.cli.main", *args],
        capture_output=True,
        text=True,
        env=env,
    )


def test_simple_signed_default_refuse_exit_11(tmp_path) -> None:
    out = tmp_path / "out.pdf"
    proc = _run_cli([str(CORPUS / "simple_signed.pdf"), "-o", str(out)])
    assert proc.returncode == 11, proc.stderr
    assert "signed" in proc.stderr.lower()
    assert "preserve" in proc.stderr.lower() or "allow-signed" in proc.stderr.lower()


def test_simple_signed_preserve_passthrough_byte_identical(tmp_path) -> None:
    out = tmp_path / "out.pdf"
    proc = _run_cli(
        [str(CORPUS / "simple_signed.pdf"), "-o", str(out), "--preserve-signatures"]
    )
    assert proc.returncode == 0, proc.stderr
    assert out.read_bytes() == (CORPUS / "simple_signed.pdf").read_bytes()


def test_simple_signed_allow_invalidation_breaks_signature(tmp_path) -> None:
    """--allow-signed-invalidation must produce output where the signature's
    byte-range hash no longer verifies. Self-signed certs don't have a trust
    chain to validate, but pyHanko CAN check whether the original byte range
    still hashes to the signed value — that's the rigorous "did we actually
    invalidate" check.

    Two acceptable outcomes:
    1. Engine recompresses + signature is broken (intact==False).
    2. Engine fully removes the signature dictionary (no signed metadata
       presented to readers; no false-trust risk).
    Test rejects only the silent-passthrough failure mode (output bytes
    equal input bytes, signature intact — engine claimed to compress but
    didn't, leaving the user thinking they have a compressed PDF when
    they don't).
    """
    from pyhanko.pdf_utils.reader import PdfFileReader

    out = tmp_path / "out.pdf"
    proc = _run_cli(
        [str(CORPUS / "simple_signed.pdf"), "-o", str(out), "--allow-signed-invalidation"]
    )
    # Compression must not refuse on signed input with the flag.
    assert proc.returncode != 11, proc.stderr
    # Two acceptable success paths: clean compression (rc=0) OR verifier
    # rejection (rc=20 EXIT_VERIFIER_FAIL). Reject any other code.
    assert proc.returncode in (0, 20), (
        f"unexpected rc={proc.returncode}; stderr=\n{proc.stderr}"
    )

    # Output bytes MUST differ from input bytes (no silent passthrough).
    in_bytes = (CORPUS / "simple_signed.pdf").read_bytes()
    if proc.returncode == 0:
        assert out.read_bytes() != in_bytes, (
            "engine claimed compression success but output is byte-identical "
            "to input — silent passthrough on a signed PDF with "
            "--allow-signed-invalidation"
        )

    if proc.returncode != 0:
        return  # verifier rejected — output may be missing/incomplete.

    with out.open("rb") as f:
        reader = PdfFileReader(f)
        sigs = list(reader.embedded_signatures)
        if not sigs:
            # Engine fully removed the signature dictionary. Acceptable;
            # users see "unsigned PDF" in their reader, no false-trust risk.
            return
        # Signature is still present in the output — verify its byte-range
        # hash no longer matches (i.e., we actually invalidated it).
        for sig in sigs:
            assert not sig.intact, (
                f"signature {sig.field_name} is intact after recompression — "
                "engine failed to invalidate"
            )


def test_preserve_and_allow_mutually_exclusive(tmp_path) -> None:
```

```python
"""Deterministically generate decompression-bomb fixtures.

Run: python scripts/generate_bomb_corpus.py tests/corpus/bombs/

Outputs:
  huge_page_dimensions.pdf  — 60000x20000 pt MediaBox (1.2B pixel raster at 72 DPI)
  xref_loop.pdf             — xref entry pointing to itself
  objstm_explosion.pdf      — 10,000-object Object Stream

The script runs in CI as a regression check; if a regenerated file
differs from the checked-in version, CI fails (catches non-deterministic
generation). Each fixture is < 1 MB and checked in. The 100 MB
length-mismatch fixture is generated on-demand (too big to commit).
"""

from __future__ import annotations

import argparse
import struct
from pathlib import Path


def _huge_page_dimensions(out: Path) -> None:
    """Tiny PDF whose only page declares /MediaBox [0 0 60000 20000].

    A naïve rasterizer at 72 DPI would allocate a 1.2B-pixel buffer.
    Pillow's MAX_IMAGE_PIXELS guard refuses; the input-policy gate
    catches it earlier as oversize-dimensions.
    """
    body = (
        b"%PDF-1.7\n"
        b"1 0 obj<</Type/Catalog/Pages 2 0 R>>endobj\n"
        b"2 0 obj<</Type/Pages/Count 1/Kids[3 0 R]>>endobj\n"
        b"3 0 obj<</Type/Page/Parent 2 0 R/MediaBox[0 0 60000 20000]>>endobj\n"
    )
    xref_offset = len(body)
    xref = (
        b"xref\n0 4\n0000000000 65535 f\n"
        + b"0000000009 00000 n\n"
        + b"0000000056 00000 n\n"
        + b"0000000108 00000 n\n"
    )
    trailer = (
        b"trailer<</Size 4/Root 1 0 R>>\n"
        b"startxref\n" + str(xref_offset).encode() + b"\n%%EOF\n"
    )
    out.write_bytes(body + xref + trailer)


def _xref_loop(out: Path) -> None:
    """A PDF whose xref table points to itself (cycle)."""
    body = b"%PDF-1.7\n%intentional xref-loop bomb\n"
    xref_offset = len(body)
    # Forge an xref that claims an object at its own offset.
    xref = (
        b"xref\n0 1\n"
        + f"{xref_offset:010d} 65535 n\n".encode()
    )
    trailer = (
        b"trailer<</Size 1/Root 1 0 R>>\n"
        b"startxref\n" + str(xref_offset).encode() + b"\n%%EOF\n"
    )
    out.write_bytes(body + xref + trailer)


def _objstm_explosion(out: Path) -> None:
    """ObjStm with 10,000 trivial objects.

    Real-world bombs use this to balloon parser memory. We don't need
    nested arrays — 10k objects alone trips the input-policy gate.
    """
    parts: list[bytes] = [b"%PDF-1.7\n"]
    for i in range(1, 10_001):
        parts.append(f"{i} 0 obj<</X {i}>>endobj\n".encode())
    body = b"".join(parts)
    xref_offset = len(body)
    xref = b"xref\n0 10001\n0000000000 65535 f\n" + b"".join(
        b"0000000000 00000 n\n" for _ in range(10_000)
    )
    trailer = (
        b"trailer<</Size 10001/Root 1 0 R>>\n"
        b"startxref\n" + str(xref_offset).encode() + b"\n%%EOF\n"
    )
    out.write_bytes(body + xref + trailer)


def _length_mismatch(out: Path) -> None:
    """Stream object with /Length 2_000_000_000 but ~100 MB actual data.

    Generated on-demand only (too big to commit). Triggers
    parser refusal when the lib materializes the stream and notices
    the declared/actual length divergence.
    """
    declared = 2_000_000_000
    actual = b"\x00" * (100 * 1024 * 1024)
    body = b"%PDF-1.7\n"
    obj = (
        f"1 0 obj<</Length {declared}>>stream\n".encode()
        + actual
        + b"\nendstream\nendobj\n"
    )
    body += obj
    xref_offset = len(body)
    xref = b"xref\n0 2\n0000000000 65535 f\n0000000009 00000 n\n"
    trailer = (
        b"trailer<</Size 2/Root 1 0 R>>\n"
        b"startxref\n" + str(xref_offset).encode() + b"\n%%EOF\n"
    )
    out.write_bytes(body + xref + trailer)


def main() -> None:
    p = argparse.ArgumentParser()
    p.add_argument("out_dir", type=Path)
    p.add_argument("--include-large", action="store_true",
                   help="Also generate length_mismatch.pdf (~100 MB).")
    args = p.parse_args()
    args.out_dir.mkdir(parents=True, exist_ok=True)
    _huge_page_dimensions(args.out_dir / "huge_page_dimensions.pdf")
    _xref_loop(args.out_dir / "xref_loop.pdf")
    _objstm_explosion(args.out_dir / "objstm_explosion.pdf")
    if args.include_large:
        _length_mismatch(args.out_dir / "length_mismatch.pdf")
    print(f"wrote bomb fixtures to {args.out_dir}")


if __name__ == "__main__":
    main()
```

```python
def _run(input_pdf: Path, tmp_path: Path) -> tuple[int, float, str]:
    import os

    env = os.environ.copy()
    env["HANKPDF_SKIP_ENV_CHECK"] = "1"
    out_pdf = tmp_path / "out.pdf"
    t0 = time.monotonic()
    proc = subprocess.run(
        [sys.executable, "-m", "hankpdf.cli.main", str(input_pdf), "-o", str(out_pdf)],
        capture_output=True,
        text=True,
        env=env,
        timeout=60,
    )
    elapsed = time.monotonic() - t0
    return proc.returncode, elapsed, proc.stderr
```
